# 🧩 orchestration-efficiency-skill

## 🎯 Purpose

Keep orchestration minimal, deterministic, and context-efficient.

---

## Responsibilities

- minimal skill injection
- anti-greedy retrieval
- anti-root-context enforcement
- focused dispatch orchestration

---

## Rules

- inject ONLY task-related skills
- prefer memory-driven selection
- keyword fallback only if required
- remove duplicated/overlapping skills
- prefer focused skills over broad architecture dumping

---

## Retrieval Rules

- load ONLY relevant memory keys
- avoid full project-memory loading
- avoid unrelated runtime/security retrieval
- prefer minimal invariant context

---

## Skill Injection Limits

Preferred:
- 2-5 focused skills

Avoid:
- full skill-pack injection
- unrelated lifecycle/security skills
- broad architecture context dumping

---

## Dispatch Rules

Builder:
- implementation only

Reviewer:
- architecture/runtime validation only

Security:
- realtime/security validation only

Memory:
- persistence only

---

## Forbidden

- inject all skills blindly
- route unrelated architecture context
- duplicate orchestration instructions
- use memory as commit history

---

## Goal

Minimal, focused, and deterministic multi-agent orchestration.
